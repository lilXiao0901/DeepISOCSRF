這是一個用於偵測 CSRF (跨站請求偽造) 攻擊的機器學習專案。此專案採用了一套自動化且穩健的流程，結合了孤立森林 (Isolation Forest) 進行特徵工程與一維卷積神經網路 (1D CNN) 進行分類，旨在從網路請求特徵中高效且準確地識別出惡意請求。

整個流程被封裝在一個 Python 腳本中，涵蓋了從資料前處理、自動化特徵篩選、處理類別不平衡、模型訓練、交叉驗證到最終性能視覺化的完整管線。

✨ 專案特色
混合式模型架構: 創新地使用孤立森林演算法產生「異常分數」作為新特徵，再將其與原始特徵一同輸入 1D CNN 模型，結合了無監督異常偵測與監督式深度學習的優點。

自動化特徵篩選: 透過 L1 正規化 (Lasso) 系統性地評估不同數量特徵組合的性能，自動找出最佳的特徵子集，減少人工介入並提升模型效率。

先進的資料平衡技術: 採用 Borderline-SMOTE 演算法來處理嚴重的類別不平衡問題。此方法專注於在決策邊界附近生成新的攻擊樣本，比傳統 SMOTE 更能幫助模型學習如何區分困難樣本。

Focal Loss 損失函數: 使用專為類別不平衡設計的 Focal Loss，讓模型在訓練過程中更專注於學習難以分類的樣本，有效提升對少數類（CSRF 攻擊）的偵測能力。

穩健的性能評估: 使用 10 折層化交叉驗證 (10-Fold Stratified Cross-Validation) 對模型進行評估，確保性能指標的穩定性與可靠性，避免因單次資料劃分造成的偶然性。

豐富的視覺化報告: 自動產生十多種視覺化圖表，包括資料分佈、特徵重要性、t-SNE 降維、孤立森林分數分佈、混淆矩陣、ROC 曲線及完整的模型性能指標圖，使分析過程與結果一目了然。

開箱即用: 腳本內建假資料生成功能。即使您沒有 mitch.csv 資料集，也能直接運行程式碼，體驗完整的機器學習流程。

⚙️ 運作流程
腳本的執行流程主要分為以下幾個階段：

資料載入與前處理:

讀取 mitch.csv 資料集。

使用 99% 分位數過濾極端離群值，降低其對模型的影響。

將標籤（'y'/'n'）轉換為數值格式（1/0）。

階段一：系統化特徵篩選:

使用帶有 L1 懲罰項的邏輯斯迴歸，對 5 到 11 個特徵的不同組合進行快速性能測試。

以 F1-Score 作為評估指標，繪製性能曲線圖，並自動選出表現最佳的特徵數量 k。

同時，視覺化所有 11 個候選特徵的重要性排序。

階段二：資料增強與平衡:

根據階段一選出的 k 個最佳特徵篩選資料。

對多數類別（正常請求）的樣本加入微量雜訊，進行資料增強。

使用 Borderline-SMOTE 演算法對少數類別（CSRF 攻擊）進行過採樣，使資料達到平衡狀態。

階段三：10 折交叉驗證與模型訓練:

將平衡後的資料切分為 10 折，進行交叉驗證。

在每一折中：

訓練孤立森林: 將訓練集資料餵給孤立森林，學習正常請求的模式。

產生異常分數特徵: 使用訓練好的孤立森林模型，為訓練集與測試集的所有樣本計算「異常分數」，並將此分數作為一個全新的特徵。

訓練 1D CNN: 將原始特徵與新的「異常分數特徵」合併，輸入 1D CNN 模型進行訓練。模型使用 Focal Loss 作為損失函數，並配置了提早停止 (Early Stopping) 和學習率動態調整 (ReduceLROnPlateau) 機制以防止過擬合。

收集結果: 儲存每一折的訓練歷史、孤立森林預測結果、以及 CNN 的預測結果。

階段四：匯總報告與視覺化:

將 10 折交叉驗證的所有測試結果匯總。

產生並顯示一系列的最終評估圖表，全面展示孤立森林和最終 CNN 模型的性能。
